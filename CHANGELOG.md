# Решение проблемы синхронизации приоритета актива и рисков

## ✅ Что было исправлено:

### 1. **Добавлена валидация/пересчет при изменении приоритета актива**
   - Новая функция `updateAssetPriority(assetId, newPriority)`
   - При изменении приоритета актива автоматически пересчитываются:
     - Все риски, привязанные к активу
     - Исходные баллы риска (если изменился приоритет)
     - Остаточные риски (если привязана защитная мера)
   - Дашборд обновляется в реальном времени

### 2. **Добавлен интерактивный ползунок приоритета актива**
   - На вкладке "Активы": каждый актив теперь имеет ползунок для изменения приоритета (1-4)
   - До: приоритет задавался только при создании актива
   - Теперь: приоритет можно изменить после создания ✅

### 3. **Исправлено отображение связи между риском и активом**
   - До: UI показывал только ID риска (неправильно)
   - Теперь: UI показывает `"Актив: [Название актива] (приоритет: X)"`
   - Синий блок в карточке риска четко отображает связь: "Риск → Актив → Приоритет"

### 4. **Улучшена логика создания риска**
   - При добавлении риска пользователь выбирает, к какому активу его привязать
   - Риск автоматически получает приоритет выбранного актива
   - Исходный балл риска = damage × probability × priority_актива

### 5. **Улучшена логика удаления актива**
   - При удалении актива с рисками: показывается подтверждение
   - Все привязанные риски удаляются вместе с активом
   - Дашборд пересчитывается

## 📊 Как это работает:

### Сценарий: Изменение приоритета актива
```
1. Пользователь изменяет ползунок приоритета актива "База ПДн" с 3 на 4
2. Система находит все риски этого актива
3. Для каждого риска пересчитывает:
   - score = damage × probability × новый_priority
   - residualScore (если мера привязана)
4. Обновляется дашборд:
   - Общий риск до мер → увеличился
   - Общий риск после мер → может измениться
   - Экономический эффект → может измениться
5. Console логирует: "✅ Актив 'База ПДн': приоритет 3 → 4. Пересчитано рисков: 2"
```

## 🔍 Валидация:
- ✅ Приоритет актива всегда в диапазоне 1-4
- ✅ Все формулы используют актуальный приоритет актива
- ✅ При удалении актива - связанные риски удаляются
- ✅ UI всегда показывает актуальное состояние
